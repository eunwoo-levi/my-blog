# React의 특성과 Context API 소개

React는 단방향 데이터 흐름을 가지고 있다. 즉, 부모 컴포넌트에서 자식 컴포넌트로 데이 전달할 수 있지만, 역으로 자식 컴포넌트에서 부모 컴포넌트로 데이터를 전달할 수는 없다.
이러한 단방향 데이터 흐름은 React의 가장 큰 장점 중 하나이다.
하지만, 이러한 단방향 데이터 흐름은 상태 관리에 어려움을 겪게 만든다. 특히, 여러개의 컴포넌트가 동일한 상태를 공유해야 하는 경우에는 더욱 그렇다.

단방향 성격때문에 일반적인 React 애플리케이션에서 데이터는 위에서 아래로 (즉, 부모로부터 자식에게) props를 통해 전달되지만, 애플리케이션 안의 여러 컴포넌트들에 전해줘야 하는 props의 경우 (예를 들면 선호 로케일, UI 테마) 이 과정이 번거로울 수 있다.
context를 이용하면, 트리 단계마다 명시적으로 props를 넘겨주지 않아도 많은 컴포넌트가 이러한 값을 공유하도록 할 수 있다.

context는 React 컴포넌트 트리 안에서 전역적(global)이라고 볼 수 있는 데이터를 공유할 수 있도록 고안된 방법이다.
context를 사용하면 중간에 있는 Component(엘리먼트)들에게 props를 넘겨주지 않아도 된다.

즉, 컴포넌트 안에서 전역적으로 데이터를 공유할 수 있도록 나온 개념으로, React의 context는 앱의 모든 컴포넌트에서 사용할 수 있는 데이터를 전달할 때 유용하다.
사용자 로그인 데이터, 테마(theme), 언어 등 다양하게 컴포넌트 간 공유되어야할 데이터로 사용된다.

# 언제 context를 써야 할까?

React에서 컴포넌트 간에 data를 전달할 때는 주로 props를 통해서 전달한다.
하지만, props를 통해서 전달하는 것은 컴포넌트가 깊어질수록 우리가 흔히 들어본 props drilling이 발생하게 된다.
