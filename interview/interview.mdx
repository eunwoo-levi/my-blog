# 🌟 프론트엔드

## 🍀 브라우저의 렌더링 원리?

- 1. HTML 파싱 → DOM (Document Object Model) 트리 생성
  - DOM은 HTML 요소의 계층 구조를 표현
- 2.CSS 파싱 → CSSOM (CSS Object Model) 트리 생성
  - CSSOM은 각 노드에 적용될 스타일 정보를 포함
- 3. Javascript를 실행합니다.
  - `<script>` 태그가 DOM 생성 중간에 있으면, 기본적으로 파싱을 중단하고 JS를 실행
  - 이때 DOM/CSSOM을 수정할 수도 있습니다 (→ 렌더 트리 변경 가능).
  - defer, async 속성으로 파싱 차단을 줄일 수 있음.
- 4. DOM + CSSOM → 렌더 트리(Render Tree) 생성
  - `display: none` 요소는 렌더 트리에 포함되지 않습니다 (`visibility: hidden은 포함됨)
- 5. Layout (Reflow)
  - 각 렌더 트리 노드의 위치와 크기를 계산합니다.
  - 요소의 좌표계 정보 (박스 모델) 가 이 단계에서 정해짐.
- 6. Paint
  - 각 노드에 대해 색상, 텍스트, 그림자 등 실제 픽셀을 계산하여 화면에 그림
  - 복잡한 시각 효과는 Paint 비용이 큼
- 7. Composite
  - 여러 레이어가 생긴 경우 GPU를 사용해 레이어를 조합하여 최종 화면을 만듬
  - will-change, transform, opacity 등은 별도 레이어를 만들 수 있음 (Layer Promotion).
