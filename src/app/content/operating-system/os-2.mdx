# Three Main Features of Operating Systems
- **Virtualization**
ㄴ CPU Virtualization (Scheduling, Process…) 
ㄴ Memory Virtualization (Paging, Address Translation…)

Virtualization 를 쉽게 말하면, 
나의 system은 하나밖에 없지만 사용자에겐 무한개의 resource이 있는 것 같은 illusion을 주는 것.


- **Concurrency**
ㄴ Threads, Locks and semaphores, Deadlocks…

- **Persistence**
ㄴ File system, I/O Devices, Journaling…

커널 (OS) 영역:

메모리의 가장 높은 주소에 위치합니다.
운영체제의 핵심 부분이 여기에 로드됩니다.
사용자 프로그램은 이 영역에 직접 접근할 수 없습니다.


스택 영역:

커널 바로 아래에 위치합니다.
로컬 변수와 함수 호출 정보를 저장합니다.
컴파일러에 의해 자동으로 관리됩니다.


힙 영역:

동적으로 할당되는 메모리 영역입니다.
프로그래머가 직접 관리해야 합니다 (메모리 누수 방지 등).


데이터 영역:

전역 변수와 정적 변수가 저장됩니다.
프로세스 시작 시 초기화됩니다.


텍스트 또는 코드 영역:

실행 가능한 프로그램 코드가 저장됩니다.
읽기 전용이며 실행 가능합니다.


시스템 예약 영역:

메모리의 가장 낮은 주소에 위치합니다.
운영체제에 의해 특별한 용도로 사용됩니다.

# "KB" is 2^10 byte ? or 10^3 byte? 

- 2^10 바이트 (1024 바이트):

이는 전통적인 컴퓨터 과학에서의 정의입니다.
2의 거듭제곱을 기반으로 하며, 이를 "이진 킬로바이트" 또는 "키비바이트(KiB)"라고 부릅니다.
1 KiB = 2^10 = 1024 바이트


- 10^3 바이트 (1000 바이트):

이는 국제단위계(SI)에 따른 정의입니다.
10의 거듭제곱을 기반으로 하며, 공식적으로 "킬로바이트(KB)"로 불립니다.
1 KB = 10^3 = 1000 바이트

<br/>
<br/>
이러한 차이로 인해 혼란을 피하고자, 국제전기전자기술자협회(IEEE)와 국제전기통신연합(ITU)에서는 다음과 같은 용어를 제안했습니다:

KiB (키비바이트): 2^10 바이트
KB (킬로바이트): 10^3 바이트

결론적으로, "KB"가 2^10 바이트인지 10^3 바이트인지는 컨텍스트에 따라 다릅니다. 정확한 의미를 파악하기 위해서는 사용되는 맥락을 고려해야 하며, 명확성을 위해 KiB와 KB를 구분해서 사용하는 것이 좋습니다.

# Virtualizing Memory


- The OS maps address space onto the physical memory.
즉, 운영 체제는 각 프로세스가 사용하는 가상 주소 공간을 물리적 메모리의 실제 주소에 매핑합니다. 이는 메모리 가상화의 기본 원리로, 프로세스는 자신의 가상 주소를 사용하고, 운영 체제는 이 가상 주소를 물리적 메모리의 실제 주소로 변환합니다.!
	- A memory reference within one running program does not affect the address space of other processes -> Protection
	- Physical memory is a shared and limited resource, managed by the OS
▫	(c.f.) cache memory

# Concurrency

- Concurrent access of shared resources is subtle and error-prone
	- The OS is juggling **many things at once**, first running one process, then another, and so forth
	- Modern **multi-threaded programs** also exhibit the concurrency problem
	- Need **protection**, lock or other synchronization methods


***

# Persistence

- Devices such as DRAM store values in a **volatile**(휘발성)
- Hardware and software are needed to store data **persistently**
	-Hardware: I/O device such as a hard drive (HDDs) & solid-state drives(SSDs)
	-Software:
		▫	**File system** manages the disk.
		▫	File system is responsible for storing any files the user creates.


- (Sudden) Power off or system crash …

- I/O device, disk, RAIDs, file systems, …
